# Go 활용한 Banking & Board Web Service

안전한 백엔드 시스템의 구조를 이해하고, 실질적인 웹 보안 취약점 방어와 금융 트랜잭션 무결성을 실습하는 프로젝트입니다.

---

## 실행 방법 (How to Run)

### 1. 환경점검
* `main.go` (서버 소스 코드)
* `app.db` (SQLite 데이터베이스 파일)
* `schema.sql`, `seed.sql` (DB 초기화 및 샘플 데이터 스크립트)
* `/static` 폴더 (HTML, JS, CSS 등 정적 리소스)

### 2. 의존성 설치 및 서버 실행
터미널을 열고 프로젝트 루트 경로에서 아래 명령어를 입력합니다.
```bash
./cmd/server
```
서버 실행 (포트 8089) - 포트 충돌로 인해 8089 포트를 사용하였습니다.

```bash
go run main.go
```
log 폴더 및 api.log 파일의 생성을 확인.

## 세부 작업 내용

스켈레톤 코드의 9개의 Todo에 대하여, 보안성을 고려하여 기능을 구현했습니다.
기본적으로 Prepared statement를 통해 사용자 입력값이 그대로 명령어로 들어가는 것을 막았습니다. 


### 1. 계정 및 인증 시스템
* **[Todo 1] register**: `unicode` 패키지를 활용하여 비밀번호 정책(8자 이상, 숫자 포함)을 검증하는 `hasDigit()` 함수를 구현, 적용했습니다.
* **[Todo 2] logout**: 로그아웃 시 `SessionStore`에서 토큰을 즉시 삭제하고, 클라이언트 쿠키를 만료(`MaxAge: -1`)시키는 로직을 구현했습니다.
* **[Todo 3] withdraw**: 탈퇴 시 로그인 상태이더라도, 비밀번호를 한 번 더 입력받아 일치 여부를 확인하는 2차 검증 로직을 구현했습니다.

### 2. 금융 서비스 및 트랜잭션
* **[Todo 4] Deposit**: 입금 기능을 구현하였습니다.
* **[Todo 5] Withdraw**: SQL 조건절(`WHERE balance >= ?`)을 활용하여, 별도의 조회 쿼리 없이 한 번의 `Update`로 잔액 부족 여부를 체크하였습니다.
* **[Todo 6] Transfer 트랜잭션**: `db.Begin()`을 통한 트랜잭션 처리를 구현했습니다. (송신자의 잔액 차감과 수신자의 잔액 증액이 모두 성공했을 때만 `Commit`. 예외 발생 시 `Rollback`)

### 3. 게시판 관리 및 접근 제어 (Board & Access Control)
* **[Todo 7] 게시글 목록 조회**: `JOIN` 쿼리를 사용하여 `posts`와 `users` 테이블을 연계하여, 작성자의 이름과 이메일 정보를 추출했습니다.
* **[Todo 8] 소유권 기반 게시글 생성**: 세션에서 추출한 `user.ID`를 게시글의 `owner_id`로 자동 할당하여 작성자 정보 위조를 방지했습니다.
* **[Todo 9] 게시글 상세 조회 및 삭제**:
    * `c.Param("id")`를 이용해 가변 경로의 게시글 ID를 처리했습니다.
    * 삭제 시 `WHERE id = ? AND owner_id = ?` 조건을 강제하여, 본인의 게시물만 삭제할 수 있도록 하였습니다.

### 4. 시스템 운영 환경 (Infrastructure)
* **로깅 미들웨어**: 모든 HTTP 요청에 대해 메서드, 경로, 상태 코드, 지연 시간을 로깅 미들웨어를 구축했습니다.
* **로그 파일 시스템**:`log/api.log` 파일에 모든 접속을 자동 기록하는 시스템을 구축했습니다.
